# Dev Log — Math Tutor

## June 15, 2026

**What I built:**
Added image input to the tutor. Students can now include a screenshot path anywhere in their message (e.g. `I don't get this problem /Users/.../sample.png teach me`) and the program automatically detects it, reads the image, converts it to base64, and sends it to the AI along with the question. Also switched from DeepSeek to Gemini after discovering DeepSeek V4 doesn't support image input.

**What I learned:**
- Why base64 is needed: the AI runs on a remote server and can't access local files, so images must be encoded as strings and sent via JSON
- The differences between OpenAI and Gemini API formats: `content` → `parts`, `image_url` → `inline_data`, `"assistant"` → `"model"`, system prompt passed via `config={"system_instruction": ...}`
- How to scan user input for a file path using `os.path.exists()` inside a `for` loop, regardless of where in the sentence the path appears

**Problems I solved:**
- DeepSeek V4 (Pro and Flash) does not support image input — switched to Gemini
- Accidentally leaked a Gemini API key in chat — deleted and regenerated immediately
- `f.read` without parentheses returns the function object, not the data
- `os.path.exists(None)` throws an error — added `if image_path:` check first
- Gemini free tier has daily token limits; hitting quota errors with image-heavy requests

**Next:**

- Build RAG manually: read PDF → chunk → vector search → generate new problems
- SQL wrong-answer notebook
- Refactor with LangChain
- Streamlit UI (image upload via button, no path typing needed)

## Project Overview

**Math Tutor** is a Socratic AI tutor that guides students through math problems step by step, rather than giving full solutions upfront.

**Core problem it solves:**
Standard AI gives walls of text. Practice problems generated by AI are too easy and too generic to prepare for real exams. This project fixes both.

**Planned features:**
- [x] Multi-turn conversation
- [x] Socratic system prompt
- [x] Image / screenshot input
- [ ] PDF input
- [ ] RAG-based question bank from past exams
- [ ] Generate new questions matched to real exam difficulty
- [ ] Mistake tracker (SQL)
- [ ] Web interface (Streamlit)

**Tech stack:** Python · Gemini API · LangChain · ChromaDB · SQLite · Streamlit
